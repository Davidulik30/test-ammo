Добавление лобби и выбора танка

Файлы, добавленные в проект:

- `lobby/network_manager.gd` — основной сетевой менеджер (host/join, обработка списка игроков, старт игры и синхронизация)
- `lobby/match_state.gd` — простой контейнер для передачи данных игроков между сценами (рекомендуется сделать Autoload с именем `MatchState`)
- `lobby/lobby.gd` — скрипт для сцены лобби (UI с полями IP/Port и кнопками Host/Join/Quit)
- `lobby/tank_select.gd` — скрипт для сцены выбора танка (кнопки для выбора танка, список игроков, старт игры для хоста)

Краткая инструкция по подключению в Godot:

1) Создай сцену `res://lobby/lobby.tscn` (Control) с дочерними узлами:
   - `LineEdit` именем `IPLine`
   - `LineEdit` именем `PortLine`
   - `Button` именем `HostButton`
   - `Button` именем `JoinButton`
   - `Button` именем `QuitButton`
   - `Label` именем `StatusLabel`
   - `Node` именем `NetworkManager` и прикрепи к нему `res://lobby/network_manager.gd`
   - к корню сцены прикрепи скрипт `res://lobby/lobby.gd`

2) Создай сцену `res://lobby/tank_select.tscn` (Control) с дочерними узлами:
   - `VBoxContainer` именем `TanksContainer` (сюда будут динамически добавляться кнопки танков)
   - `VBoxContainer` именем `PlayersList` (список подключённых игроков)
   - `Button` именем `StartButton`
   - прикрепи к корню `res://lobby/tank_select.gd`

3) Рекомендуется добавить `res://lobby/match_state.gd` в Autoload с именем `MatchState` (Project Settings -> AutoLoad). Тогда `network_manager` при загрузке уровня запишет туда `players` и локальный peer id.

4) Тестирование локально:
   - Запусти одну копию игры и нажми Host (порт 7777 по умолчанию).
   - Запусти вторую копию игры (на той же машине) и нажми Join с IP `127.0.0.1` и порт `7777`.

5) Как передаётся информация в `artilerry_scene.tscn`:
   - Когда хост нажимает Start и все игроки выбрали танки, менеджер делает rpc вызов `client_load_scene(scene_path, players_dict)`.
   - Скрипт сохраняет `players` в `MatchState.players` (если есть Autoload) или в метаданные `root` как запасной вариант.
   - Затем локально вызывается `change_scene_to_file(scene_path)`.
